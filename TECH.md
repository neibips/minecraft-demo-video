Создай полноценную браузерную voxel-игру в стиле Minecraft под названием Minecraft 2.
Технологический стек
Обязательно используй:
* TypeScript
* Vite
* Babylon.js
* Web Workers
* IndexedDB
  Проект должен быть современным, модульным, хорошо структурированным и готовым к расширению. Код должен быть чистым, типизированным и разделённым на логические подсистемы.
  Общая цель
  Нужно реализовать веб-версию Minecraft-подобной игры, работающей прямо в браузере, с процедурной генерацией мира, сохранением мира и прогресса игрока, крафтом, мобами, предметами, базовой физикой, анимациями, инвентарём и качественным UI.
  Игра должна ощущаться как качественная sandbox-survival voxel-игра, максимально близкая по основным взаимодействиям к Minecraft, но адаптированная под веб.

Источники данных
В проекте уже есть данные, которые нужно обязательно использовать.
Блоки
* Список блоков находится в assets/blocks.txt
* Текстуры блоков находятся в папке textures/
  Сущности
* Мобы и сущности перечислены в entities.json
  Предметы
* Предметы перечислены в items.json
  Рецепты
* Рецепты крафта перечислены в reciepes.json
  Важно
  Нужно вчитываться в описания и поля этих файлов, потому что там есть важные уточнения по логике, свойствам, поведению, категориям и взаимосвязям объектов. Нельзя просто игнорировать содержимое файлов и делать заглушки. Структуры из этих файлов должны стать основой игровых систем:
* реестра блоков
* реестра предметов
* реестра мобов
* системы крафта
* визуального отображения
* игровых свойств

Основные требования к игре
1. Мир и генерация
   Реализуй процедурную генерацию мира с разделением на чанки.
   Требования к генерации:
* мир бесконечный или практически бесконечный по чанкам
* генерация должна быть детерминированной по seed
* чанки должны загружаться и выгружаться динамически вокруг игрока
* генерация и тяжёлые вычисления должны быть вынесены в Web Workers
* должна быть генерация:
    * травы
    * земли
    * камня
    * песка
    * воды
    * деревьев
    * цветов
    * возвышенностей и неровностей рельефа
    * мобов по биомам/условиям, если это возможно из данных
      Дополнительно желательно:
* простые биомы, например:
    * равнина
    * лес
    * пляж
    * водоём
    * Горы
* разные уровни высоты
* береговая линия
* подземный слой из камня
  Чанки:
* отдельная структура данных для хранения чанков
* кэш чанков
* сохранение изменённых чанков
* повторное восстановление чанков из IndexedDB
* если чанк уже был изменён игроком, брать данные из сохранения, а не генерировать заново

2. Сохранение игры
   Все данные игры должны сохраняться в IndexedDB.
   Нужно сохранять:
* seed мира
* все изменённые чанки
* позицию игрока
* здоровье игрока
* инвентарь
* быстрый доступ / hotbar
* выбранный слот
* размещённые и сломанные блоки
* состояние мобов, если реализовано
* настройки игры
* список сохранений или хотя бы одно основное сохранение
  Поведение:
* автосохранение
* ручное сохранение через меню
* загрузка игры из меню
* корректное восстановление состояния после перезагрузки страницы
  IndexedDB:
  Сделай нормальную обёртку над IndexedDB:
* версии схемы
* отдельные store для мира, игрока, метаданных, настроек
* безопасная сериализация данных
* обработка ошибок
* возможность расширения формата сохранений

3. Игрок и управление
   Игрок должен иметь привычные действия, как в Minecraft.
   Обязательные действия:
* ходьба: WASD
* прыжок: Space
* приседание: Shift
* бег: Ctrl или Shift, выбери понятную схему
* управление камерой мышью
* pointer lock
* разрушение блоков
* установка блоков
* выбор слота hotbar
* подбор предметов
* открытие инвентаря на E
  От первого лица:
* камера от первого лица
* игрок должен видеть свою руку
* в руке должен отображаться активный предмет
* должна быть анимация руки:
    * в покое
    * при ударе / разрушении
    * при установке блока
    * желательно при использовании предмета

4. Интерфейс
   Сделай качественный UI, аккуратный, современный, визуально приятный и читаемый.
   HUD:
   Внизу экрана должны быть:
* полоска или сердечки здоровья
* hotbar / быстрый доступ к предметам
* подсветка выбранного слота
* при необходимости индикатор количества предметов в слотах
  Инвентарь:
  На клавишу E открывается инвентарь.
  Инвентарь должен включать:
* сетку инвентаря
* hotbar
* слот(ы) экипированного / активного предмета, если нужно
* область крафта 2x2 (4 ячейки)
* слот результата крафта
  Меню:
  Нужно реализовать:
* главное меню
* экран загрузки / создания мира
* меню паузы
* меню сохранения / загрузки
* настройки
* экран смерти, если есть здоровье и смерть
  UX:
* drag & drop предметов
* разделение стаков хотя бы на базовом уровне
* tooltip с названием предмета/блока
* понятная навигация мышью и клавиатурой

5. Крафт
   Нужно реализовать систему крафта на основе reciepes.json.
   Требования:
* загрузка рецептов из файла
* Расположить ресурсы поштучно в ячейку
* проверка совпадений в сетке 2x2
* отображение результата крафта
* перенос результата в инвентарь
* уменьшение количества входных ресурсов
* поддержка stackable items
  Важно:
* структура рецептов должна быть прочитана из файла, а не захардкожена
* учитывать все важные поля и уточнения из reciepes.json

6. Блоки и взаимодействие с миром
   Используй блоки из assets/blocks.txt и текстуры из textures/.
   Требования:
* текстурированный voxel-мир
* корректное сопоставление блока и текстуры
* поддержка разных типов блоков:
    * твёрдые
    * прозрачные
    * полупрозрачные
    * вода
    * декоративные
    * растения
* выделение блока, на который смотрит игрок
* raycast / picking для взаимодействия
* размещение блока на соседней грани
* разрушение блока с анимацией/эффектом, если возможно
* выпадение предмета после разрушения, если это предусмотрено логикой
  Важно:
  если у блока в данных есть особые свойства, использовать их в логике:
* прочность
* прозрачность
* коллизия
* emit light, если есть
* категория
* дроп
* поведение при размещении

7. Предметы
   Используй items.json как главный источник данных о предметах.
   Реализовать:
* реестр предметов
* иконки / визуальное отображение
* стеки предметов
* поднятие предметов
* размещение блока предметом, если это block item
* использование предметов, если логика позволяет
* отображение активного предмета в руке
  Учитывать:
* названия
* типы
* стаки
* связи с блоками
* служебные поля из JSON

8. Мобы и сущности
   Используй entities.json.
   Нужно:
* создать базовую систему сущностей
* реализовать хотя бы несколько мобов на основе данных
* спавн мобов в мире
* для годзилы сгенерировать модель и текстуру самостоятельно.
* простое поведение:
    * idle
    * перемещение
    * поворот
    * следование простым правилам
* базовые анимации мобов
  Минимум:
* система компонентов или классов для сущностей
* позиция, скорость, здоровье, состояние
* рендер
* коллизии с миром
* простая логика AI
  Желательно:
* пассивные мобы
* враждебные мобы
* получение урона
* смерть и дроп

9. Физика
   Физика должна быть похожа на Minecraft.
   Обязательно:
* гравитация
* прыжок
* коллизии игрока с блоками
* движение по земле
* запрет прохождения сквозь твёрдые блоки
* обработка лестниц/воды по возможности
* Урон от падения с высоты
* более медленное движение в воде, если вода реализована как особая среда
  Важно:
* поведение должно быть стабильным
* без застреваний в блоках
* с нормальной обработкой краёв, ступенек и падения

14. Игровой цикл
    Минимальный игровой цикл должен работать так:
1. Игрок создаёт или загружает мир
2. Мир генерируется по seed
3. Игрок появляется в безопасной точке
4. Игрок исследует мир
5. Добывает блоки
6. Подбирает предметы
7. Использует hotbar
8. Открывает инвентарь
9. Крафтит предметы через 2x2 сетку
10. Строит и изменяет мир
11. Весь прогресс сохраняется в IndexedDB
12. При следующем запуске всё восстанавливается
